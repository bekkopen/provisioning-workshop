Intro til provisjoneringsrammeverk

Hva er provisjoneringsrammeverk?

  • Automatisert oppsett av servere
  • Konfigurasjon som kode

Eksempler

  • Lage brukere
  • Generere configfiler
  • Installere programvare
  • Starte/stoppe/restarte prosesser
  • Sette opp avhengigheter mellom operasjoner

Pros

  • I kildekodekontroll.
  • Likere miljøer.
  • Selvdokumeterende (det er kode!)
  • Refactoring
  • Repeterbart (no need for manual rollback)
  • Deterministisk
  • Fjerner manuelle steg
  • Raskt å konfigurere opp et nytt miljø
  • Enklere å teste serveroppsett

Cons

  • Uvant for "klassiske driftere"
  • Uvant med drift for utviklere
  • Oppstartskostnad
    • Opplæring
    • Migrering
  • Krever en del av organisasjonen
Organisasjon: kan være silobasert.

Ordforklaring

  • Master/Slave
  • SSH
  • Imperativt versus deklarativt

Imperativt


#!/bin/bash
#Add more steps to check if
#present and so on...

yum install vim

echo "set nocp" > /etc/vim/vimrc

Deklarativt


package { "vim":
  ensure => present,
}

file { "/etc/vim/vimrc":
  content => "set nocp",
}

Oppgaven

  • An app user 'devops', with:
    • 'devops123' as password
    • a home directory
    • /bin/bash as the default shell
  • an init script
  • nginx with a reverse proxy using example configuration
